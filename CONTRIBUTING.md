# Contributing to txt.wwew.tech

Thank you for your interest in improving this project.

## Quick Start

1. Fork the repository.
2. Create a branch from `master`:
   `git checkout -b feat/short-description`
3. Install dependencies:
   `npm ci`
4. Run checks locally:
   `npm run lint`
   `npm run test`
5. Open a pull request with a clear description.

## Development Guidelines

* Keep changes focused and minimal.
* Add or update tests for behavior changes.
* Preserve existing code style and architecture.
* Update README/docs when introducing user-visible changes.

## Commit Messages

Use concise imperative messages, for example:

* `fix: prevent crash on empty markdown`
* `feat: add mobile preview modal keyboard support`

## Pull Request Checklist

* [ ] I read relevant docs and existing code before making changes.
* [ ] I ran lint and tests locally.
* [ ] I added/updated tests where needed.
* [ ] I described the change and its motivation.

## Reporting Bugs and Suggesting Features

Please use GitHub Issues and include:

* Reproduction steps
* Expected behavior
* Actual behavior
* Environment details (OS, browser, versions)
