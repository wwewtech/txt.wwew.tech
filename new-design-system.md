# Vercel Design System — Глубокий разбор UI/UX

---

## I. ЦВЕТОВАЯ СИСТЕМА (Design Tokens)

### Структура — 10-уровневая шкала на каждый цвет

Система построена на **HSL-переменных**, которые автоматически переключаются между светлой и тёмной темой через `:root` / `.dark-theme`:

```css
--ds-[color]-[100..1000]-value → hsla(hue, sat%, light%, alpha)
```

Каждый смысловой цвет имеет **10 ступеней** — от почти прозрачного до почти чёрного/белого:

| Цвет | 100 (самый светлый) | 900 (насыщенный) | 1000 (темнейший) |
|---|---|---|---|
| **Gray** | `hsl(0,0%,95%)` | `hsl(0,0%,30%)` | `hsl(0,0%,9%)` |
| **Blue** | `hsl(212,100%,97%)` | `hsl(211,100%,42%)` | `hsl(211,100%,15%)` |
| **Red** | `hsl(0,100%,97%)` | `hsl(358,66%,48%)` | `hsl(355,49%,15%)` |
| **Green** | `hsl(120,60%,96%)` | `hsl(133,50%,32%)` | `hsl(128,29%,15%)` |
| **Amber** | `hsl(39,100%,95%)` | `hsl(30,100%,32%)` | `hsl(20,79%,17%)` |
| **Purple** | `hsl(276,100%,97%)` | `hsl(274,71%,43%)` | `hsl(276,100%,15%)` |
| **Pink** | `hsl(330,100%,96%)` | `hsl(336,65%,45%)` | `hsl(333,74%,15%)` |
| **Teal** | `hsl(169,70%,96%)` | `hsl(174,91%,25%)` | `hsl(171,80%,13%)` |

### В тёмной теме (`dark-theme`) — ПОЛНАЯ ИНВЕРСИЯ

Все значения перераспределяются: 100 становится **тёмным** (`hsl(0,0%,10%)`), 1000 — **светлым** (`hsl(0,0%,93%)`). Это делает систему **режимо-независимой** — один и тот же токен `--ds-gray-900` — это «тёмный текст» в светлой теме и «светлый текст» в тёмной.

### Gray — нейтральная шкала (кровь системы)

```
Светлая тема:      Тёмная тема:
100 = #F2F2F2      100 = #1A1A1A  ← фон карточки
200 = #EBEBEB      200 = #1F1F1F  ← фон страницы
300 = #E6E6E6      300 = #292929
400 = #EBEBEB      400 = #2E2E2E  ← бордюры
500 = #CACACA      500 = #454545
600 = #A8A8A8      600 = #878787  ← плейсхолдер
700 = #8F8F8F      700 = #8F8F8F  ← вторичный текст
900 = #4D4D4D      900 = #A0A0A0  ← подписи
1000 = #171717     1000 = #EDEDED ← основной текст
```

### Отдельно — Alpha-канал (для теней и оверлеев)

```css
--ds-gray-alpha-100: #0000000d  /* 5% чёрного */
--ds-gray-alpha-200: #00000015  /* 8% */
--ds-gray-alpha-300: #0000001a  /* 10% */
--ds-gray-alpha-400: #00000014  /* 8%, бордюры */
--ds-gray-alpha-500: #00000036  /* 21% */
--ds-gray-alpha-900: #000000b3  /* 70% */
```

В тёмной теме — всё на белом:
`--ds-gray-alpha-400: #ffffff24` (белый 14%)

### Фоны — 2-уровневая система

```css
--ds-background-100 /* поверхность карточки/модала */
--ds-background-200 /* фон страницы/второй слой */

/* Светлая: bg-100 = #FFFFFF,    bg-200 = #FAFAFA */
/* Тёмная:  bg-100 = #0A0A0A,    bg-200 = #000000 */
```

### Цветовое применение по функции

| Цвет | Назначение |
|---|---|
| **Синий** | Интерактивные элементы, ссылки, фокус, выделение чекбоксов |
| **Красный** | Ошибки, удалённый код, деструктивные действия |
| **Зелёный** | Успех, добавленный код, статус «готово» |
| **Amber** | Предупреждения, кеш-состояния, beta-бейджи |
| **Purple** | Функции в коде, фиолетовые бейджи |
| **Pink** | Ключевые слова в коде, preview-среда |
| **Teal** | Второстепенные успех-индикаторы |

## II. ТЕНЬ-СИСТЕМА (Shadow Tokens)

Тени — не декоративные, а **семантические**: каждый слой интерфейса имеет свою тень.

```css
--ds-shadow-small    /* 0px 2px 2px rgba(0,0,0,4%) */
--ds-shadow-medium   /* 2px 2px + 8px 8px (два слоя) */
--ds-shadow-large    /* 2px 2px + 8px 16px */
--ds-shadow-tooltip  /* border + 1px + 4px 8px */
--ds-shadow-menu     /* border + 1px + 4px 8px + 16px 24px + 24px 32px */
--ds-shadow-modal    /* border + 1px + 8px 16px + 24px 32px */
```

**Принцип**: чем выше z-index, тем больше слоёв в тени. **Граница** (`0 0 0 1px`) всегда входит в тень как первый слой — это исключает необходимость в `border` у плавающих элементов.

> В тёмной теме тени **усиливаются** (больше непрозрачности), т.к. на тёмном фоне слабые тени не читаются.

## III. ПРОСТРАНСТВЕННАЯ СИСТЕМА (Spacing)

Базовая единица: **4px**. Всё кратно ей.

```css
--geist-space     = 4px    /* xs */
--geist-space-2x  = 8px    /* sm */
--geist-space-3x  = 12px
--geist-space-4x  = 16px   /* md — основной внутренний отступ */
--geist-space-6x  = 24px   /* gap между блоками */
--geist-space-8x  = 32px   /* lg */
--geist-space-16x = 64px
--geist-space-24x = 96px
--geist-space-gap = 24px   /* главный gap сетки */
--geist-space-gap-half = 12px
--geist-page-margin = 24px /* отступы от краёв страницы */
--geist-page-width  = 1200px
```

## IV. ТИПОГРАФИКА

### Шрифтовая пара

- **Geist Sans** (`font-display: swap`) — основной UI-шрифт. Геометрический гротеск.
- **Geist Mono** (`font-display: block`) — моноширинный для кода и цифр. `block` — не мигает при загрузке.

### Шкала заголовков

```css
h1: 3rem    / weight 700 / letter-spacing: -0.067rem
h2: 2.25rem / weight 600 / -0.049rem
h3: 1.5rem  / weight 600 / -0.029rem
h4: 1.25rem / weight 600 / -0.021rem
h5: 1rem    / weight 600 / -0.011rem
h6: 0.875rem/ weight 600 / -0.006rem
```

> **Ключевое**: отрицательный `letter-spacing` нарастает с размером — крупные заголовки «сжимаются», чтобы выглядеть плотнее и профессиональнее.

### Тело текста

```css
body-1: 1rem    / 400 / line-height: 1.6
body-2: 0.875rem / 400 / 1.6
small: 14px / 400
label: 12px / 500   /* метки полей, капшн */
code: 13px / Geist Mono
```

## V. КОМПОНЕНТЫ — РАЗМЕРНАЯ СИСТЕМА

Формы и кнопки — единая метрика высоты:

```css
small:   height = 32px  (--geist-form-small-height)
default: height = 40px  (--geist-form-height)
medium:  height = 40px  (--geist-space-medium)
large:   height = 48px  (--geist-form-large-height)
```

### Кнопки — X-padding по размеру

```
small:   px = 6px,  h = 32px, font = 14px
default: px = 10px, h = 40px, font = 14px
large:   px = 14px, h = 48px, font = 16px
```

### 3 типа кнопок

| Тип | Фон | Обводка | Назначение |
|---|---|---|---|
| **Primary** | `--ds-gray-1000` (чёрный) | нет | Главное действие |
| **Secondary** | `--ds-background-100` | `0 0 0 1px --ds-gray-400` | Второстепенное |
| **Tertiary** | `transparent` | нет | Ghost, вспомогательное |

### Badge — таблетки

```css
sm: h=20px, font=11px, px=6px
md: h=24px, font=12px, px=10px  /* дефолт */
lg: h=32px, font=14px, px=12px
border-radius: 9999px /* полная капсула */
```

## VIII. СОСТОЯНИЯ И ИНТЕРАКТИВНОСТЬ

### Фокус-кольцо

```css
--ds-focus-ring: 0 0 0 2px var(--ds-background-100),  /* «зазор» 2px */
                 0 0 0 4px var(--ds-focus-color);      /* синяя обводка */

/* Светлая: --ds-focus-color = --ds-blue-700 */
/* Тёмная:  --ds-focus-color = --ds-blue-900 */
```

### Transition — единая формула

```css
transition: border-color, background, color, transform, box-shadow;
/* duration: 0.15s (большинство) / 0.2s (движение) / 0.3s (модалы) */
/* timing: ease / cubic-bezier(.175,.885,.32,1.1) — «swift» для оверлеев */
```

### Анимации элементов UI

| Элемент | Анимация | Длительность |
|---|---|---|
| Меню/popover | `translateY(-5px) → 0` | 0.1s |
| Модалы | `scale(.95) + opacity` | 0.3s |
| Аккордеон | `height: 0 → auto` | 0.25s |
| Tooltip | `translateY(-10px) → 0` | 0.15s |

## IX. СИНТАКСИЧЕСКАЯ ПОДСВЕТКА КОДА

Цвета токенов встроены как CSS-переменные (Shiki):

```css
--shiki-color-text        = ds-gray-1000    /* обычный текст */
--shiki-token-comment     = ds-gray-900     /* серые комменты */
--shiki-token-keyword     = ds-pink-900     /* розовые ключевые слова */
--shiki-token-function    = ds-purple-900   /* фиолетовые функции */
--shiki-token-string      = ds-green-900    /* зелёные строки */
--shiki-token-constant    = ds-blue-900     /* синие константы */
--shiki-token-parameter   = ds-amber-900    /* янтарные параметры */
--shiki-token-punctuation = ds-gray-1000    /* нейтральные скобки */
```

## X. ИТОГ — ДИЗАЙН-ПРИНЦИПЫ, ЗАКРЕПЛЁННЫЕ В КОДЕ

| Принцип | Реализация |
|---|---|
| **Dual-mode first** | Каждый токен переключается через `:root / .dark-theme` без JS |
| **Border как shadow** | `box-shadow: 0 0 0 1px` вместо `border` у всех floating элементов |
| **4px grid** | Все отступы кратны 4px через именованные переменные |
| **No magic numbers** | Всё через CSS Custom Properties, ни одного «голого» цвета |
| **Semantic color scale** | Один и тот же токен работает в обеих темах без изменений |
| **Typographic hierarchy** | Отрицательный letter-spacing растёт вместе с font-size |
| **P3 color gamut** | `@media (color-gamut:p3)` заменяет HSL на oklch для широкого охвата |
